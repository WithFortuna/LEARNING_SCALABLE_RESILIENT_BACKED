 1. RestTemplate 사용법 공부

- 두 개의 설정 클래스 비교
    - SimpleRestTemplateConfig — 기본 new RestTemplate() (학습용 비교군)
    - PracticalRestTemplateConfig — Apache HttpClient 5 기반 실전 설정
  - 메서드별 클라이언트 분리
    - GetClient (getForObject, 경로변수, UriComponentsBuilder)
    - PostClient (postForObject)
    - PutClient, DeleteClient
    - OthersClient (HttpEntity, postForEntity, setBearerAuth)
    - ExceptionHandlingClient (HttpClientErrorException / HttpServerErrorException / RestClientException 분기)


2. 장애 핸들링을 위한 타임아웃 관리
 3종 타임아웃 + 커넥션 풀까지 모두 다루었습니다.

  ┌────────────────────────────┬─────────┬────────────────────────┐
  │       타임아웃 종류        │ 운영 값 │          의미          │
  ├────────────────────────────┼─────────┼────────────────────────┤
  │ Connection Timeout         │ 1초     │ TCP 연결 수립까지      │
  ├────────────────────────────┼─────────┼────────────────────────┤
  │ Response Timeout           │ 3초     │ 응답 수신까지          │
  ├────────────────────────────┼─────────┼────────────────────────┤
  │ Connection Request Timeout │ 1초     │ 풀에서 커넥션 획득까지 │
  └────────────────────────────┴─────────┴────────────────────────┘

  - application.yml에 외부 설정으로 분리 → HttpClientTimeoutProperties로 바인딩
  - PoolingHttpClientConnectionManager (max 100, perRoute 10)
  - evictExpiredConnections() + evictIdleConnections(60s)로 좀비 커넥션 정리



 3. 지정한 타임아웃을 테스트로 검증.
